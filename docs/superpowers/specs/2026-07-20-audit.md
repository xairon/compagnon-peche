# Audit complet — Compagnon de pêche (2026-07-20)

Audit sur 5 dimensions (5 agents spécialisés + vérifications manuelles) : correction/logique,
**données réglementaires & sanitaires**, PWA/offline/perf, accessibilité/UX, qualité/sécurité/privacy.
Verdict global : base saine (offline-first propre, 100 % local pour carnet/matériel, aucune fuite de
données personnelles, pas d'injection). Les problèmes réels trouvés ont été corrigés ; les
correctifs critiques sont vérifiés à l'écran.

## ✅ Corrigé — Bugs logiques

- **Double entrée carnet + quota doublé** (majeur) : « Ajouter au carnet » depuis « Ma prise » créait
  une entrée fantôme PUIS une seconde à l'enregistrement → 2 lignes + quota compté ×2. Corrigé
  (une seule entrée). Vérifié : 2 → 2 (formulaire) → 3.
- **Retour arrière prise protégée/invasive** (majeur, conformité) : sur une espèce protégée/invasive/
  hors-saison, le bouton « ‹ » renvoyait sur « Garder ou relâcher » (proposait de tuer une protégée /
  relâcher une invasive). Corrigé : retour contextuel vers « statut ». Vérifié.
- **Saison — jour de fermeture** : la comparaison à minuit fermait un jour trop tôt (3ᵉ dim. sept. /
  dern. dim. jan.). Corrigé (borne inclusive fin de journée).
- **Quota en date UTC** vs affichage local → bascule de journée décalée. Corrigé (`isoDay` local).

## ✅ Corrigé — PWA / hors-ligne / robustesse

- **Runtime caching** ajouté : tuiles OSM (CacheFirst) + Hub'Eau (NetworkFirst) → la carte reste
  utilisable hors-ligne sur les zones déjà consultées.
- **Police auto-hébergée** (Source Serif 4 en woff2 local) : plus de dépendance/fuite Google Fonts,
  titres rendus hors-ligne. Précachée.
- **Requêtes Hub'Eau annulables** (AbortController) : plus de course qui écrase les marqueurs, annulation
  au démontage / changement de station. Timers de la carte nettoyés.
- **Matériel** : `Promise.all` sans `.catch` → écran bloqué si IndexedDB échoue. Corrigé (`.finally`).
- **Pipeline** : dédoublonnage aussi par id + assertion runtime d'unicité des ids (dev).
- Lien `http://` (peche41) → `https://`.

## ✅ Corrigé — Accessibilité

- **Contraste** (n°1 au soleil) : `--muted` et `--faint` assombris (~4.5:1 WCAG AA).
- **Focus clavier visible** ajouté (absent partout).
- **Cibles tactiles ≥ 44px** sur les petites commandes (fermer sheet, « autour de moi », supprimer,
  fermer explication, retour étape…).
- Marqueurs carte plus gros et contrastés ; statut carte `aria-live` ; onglet actif `aria-current`.
- État vide du carnet ajouté.

## ✅ Corrigé — 2ᵉ passe (améliorations non bloquantes)

- **a11y** : `<div onClick>` → `<button>` (carte espèce, accordéon fiche avec `aria-expanded`, lignes
  Outils, tuiles Nœuds, candidats Identify, liens inline) ; `aria-label` sur boutons-icônes ;
  info-bulle reliée (`aria-describedby`) + fermeture Échap.
- **Mode gants** (`bigUI`) : propagé app-wide via `data-big` (chips, cellules, recherche, accordéons,
  sheet carte…), **persisté** (localStorage). Vérifié : chips 38 → 46 px.
- **Perf** : Leaflet **code-splitté** (`React.lazy`, chunk séparé ~43 Ko gzip, chargé uniquement sur
  la carte) ; Materiel/Guide lazy aussi.
- **PWA** : rechargement propre au changement de service worker (garde anti-boucle, pas au 1ᵉʳ
  install).
- **Robustesse** : `crypto.randomUUID()` (helper `uid`) pour carnet/matériel ; le pipeline **dérive**
  désormais les listes curées depuis `species.ts` (fin de la synchro manuelle).
